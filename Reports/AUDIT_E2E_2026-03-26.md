## 3. PROBLEME CRITICE (P0)

### P0-1: Fara Rate Limiting pe niciun endpoint
- **Impact:** Toate cele 75 endpoints vulnerabile la flood, brute force, credit abuse (AI generate)
- **Fix:** next-rate-limit sau Vercel built-in rate limiting

### P0-2: NextAuth v5 beta (5.0.0-beta.25) in productie
- **Impact:** Risc de stabilitate, breaking changes, vulnerabilitati nepatched
- **Fix:** Pin version, monitorizeaza release-uri, sau downgrade la v4 stable

### P0-3: File Upload fara limite (bulk-import)
- **Impact:** Upload multi-GB, memory exhaustion, RCE via parsing libs (pdfparse, mammoth)
- **Fix:** Max 10MB, validate magic bytes, virus scan

### P0-4: Pagina /dashboard/exams lipseste (referita in sidebar nav)
- **Impact:** Click pe Exams -> 404
- **Fix:** Creeaza pagina sau elimina link-ul din sidebar

### P0-5: Fara Error Boundaries (error.tsx)
- **Impact:** Orice eroare in component -> crash propagat global
- **Fix:** error.tsx in fiecare segment major (admin, instructor, student)

## 4. PROBLEME HIGH (P1)

### P1-1: JWT token fara validare structurala
- auth.ts linii 45-75: cast-uri fara verificare pe token.enrollments
- **Fix:** Validare Zod pe token shape

### P1-2: N+1 Query in session-engine.ts
- selectAdaptiveQuestions() face fetch per topic in loop
- **Fix:** Batch fetch cu findMany({ where: { topic: { in: [...] } } })

### P1-3: Race conditions in gamification.ts
- awardAnswerXp() fara locking - XP duplicat/pierdut la concurrent requests
- **Fix:** Row-level locking sau atomic DB operations

### P1-4: Escalation resume detection broken
- User completeaza sesiune dar L2+ notificari tot se trimit
- **Fix:** Cancel ALL pending escalations la session complete

### P1-5: WhatsApp client creat la fiecare request
- notifications/service.ts - new WhatsAppClient() la fiecare call
- **Fix:** Singleton pattern (ca in calendar.ts)

### P1-6: Group management fara ownership verification
- Instructor poate modifica/sterge grupuri create de alt instructor
- **Fix:** Check createdById === session.user.id

### P1-7: Sidebar nu e responsive pe mobile
- Latime fixa w-64, fara hamburger menu
- **Fix:** Mobile drawer cu toggle button

### P1-8: Ad URL-uri accepta javascript: / data: URIs
- XSS posibil prin ad placements
- **Fix:** Whitelist scheme (https only)

### P1-9: Silent error handling pe toate API calls
- .catch(() => {}) peste tot - utilizatorul nu stie ca a esuat ceva
- **Fix:** Toast notifications + error states

### P1-10: Calendar OAuth state fara CSRF token
- State parameter e JSON parsable dar fara token criptografic
- **Fix:** Adauga random token verificat la callback
